Sécurité et validation

Backend

  • JWT Bearer Authentication (tokens signés, expiration configurable)
  • Rate Limiting — Auth: 10 req/min, API: 60 req/min
  • CORS — Origines configurables via appsettings
  • DataAnnotations — Required, MaxLength, Range sur DTOs
  • Check Constraints — Score 1-5, pas d'auto-notation
  • Exception Middleware — Gestion centralisée des erreurs
  • Authorization — Créateur seul peut modifier/supprimer

Frontend

  • Validation inline — Erreurs sous chaque champ
  • Password criteria — Temps réel (8 chars, majuscule, chiffre)
  • Date validation — Doit être dans le futur
  • Auth guard — Redirect vers /login si non connecté
  • Axios interceptor — 401 auto-logout + redirect
  • Creator check — EditEvent vérifie le propriétaire

Base de données

  • Unique indexes — Email, Event+User, Rating
  • Foreign keys — Cascade deletes configurés
  • Enum → String conversion — Status stocké lisible
  • Timestamps auto — UpdatedAt via ChangeTracker
